# 中山文旅美食导览 App 后端项目结构设计

## 一、后端项目说明

本项目后端采用 Spring Boot 框架开发，主要为 HarmonyOS APP 提供数据接口服务。后端负责处理用户登录注册、景点数据查询、景点相册查询、美食数据查询、收藏管理、评论管理和搜索等业务功能。

系统采用前后端分离架构，HarmonyOS APP 通过 HTTP 请求访问 Spring Boot 后端接口，后端从 MySQL 数据库中查询数据，并以 JSON 格式返回给前端。

---

## 二、后端技术栈

| 技术                     | 说明      |
| ---------------------- | ------- |
| Spring Boot            | 后端主框架   |
| MyBatis / MyBatis-Plus | 数据库访问   |
| MySQL                  | 数据库存储   |
| Maven                  | 项目依赖管理  |
| Lombok                 | 简化实体类代码 |
| JWT                    | 登录认证，可选 |
| Knife4j / Swagger      | 接口文档，可选 |
| Postman / Apifox       | 接口测试工具  |

课程设计阶段可以先不做复杂权限控制，登录接口可以先返回简单 token，例如 `test-token`。如果时间充足，再加入 JWT。

## 五、各目录作用说明

### 1. common 公共工具包

用于存放统一返回结果类、常量类、异常处理类等公共代码。

主要文件：

```text
Result.java
```

Result 类用于统一后端接口返回格式，例如：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

---

### 2. controller 控制层

controller 层负责接收前端请求，并调用 service 层处理业务。

主要文件：

| 文件                      | 作用               |
| ----------------------- | ---------------- |
| UserController.java     | 用户登录、注册、获取用户信息   |
| ScenicController.java   | 景点列表、景点详情、景点相册、景点搜索 |
| FoodController.java     | 美食列表、美食详情、美食搜索   |
| FavoriteController.java | 添加收藏、取消收藏、查询我的收藏 |
| CommentController.java  | 查询评论、发布评论        |

---

### 3. entity 实体类包

entity 层用于定义数据库表对应的 Java 实体类。

主要文件：

| 文件              | 对应数据库表          |
| --------------- | --------------- |
| User.java       | user 用户表        |
| ScenicSpot.java | scenic_spot 景点表 |
| ScenicImageVO.java | scenic_image 景点相册表查询结果 |
| Food.java       | food 美食表        |
| Favorite.java   | favorite 收藏表    |
| Comment.java    | comment 评论表     |

---

### 4. mapper 数据访问层

mapper 层负责访问数据库，执行增删改查操作。

主要文件：

| 文件                  | 作用       |
| ------------------- | -------- |
| UserMapper.java     | 用户表数据库操作 |
| ScenicMapper.java   | 景点表和景点相册表数据库操作 |
| FoodMapper.java     | 美食表数据库操作 |
| FavoriteMapper.java | 收藏表数据库操作 |
| CommentMapper.java  | 评论表数据库操作 |

如果使用 MyBatis，还需要在 `resources/mapper` 下编写对应的 XML 文件。

---

### 5. service 业务层

service 层用于处理具体业务逻辑，例如登录判断、收藏判断、搜索处理等。

主要文件：

| 文件                   | 作用     |
| -------------------- | ------ |
| UserService.java     | 用户业务接口 |
| ScenicService.java   | 景点和景点相册业务接口 |
| FoodService.java     | 美食业务接口 |
| FavoriteService.java | 收藏业务接口 |
| CommentService.java  | 评论业务接口 |

---

### 6. service.impl 业务实现层

service.impl 层用于实现 service 接口中的方法。

主要文件：

| 文件                       | 作用     |
| ------------------------ | ------ |
| UserServiceImpl.java     | 用户业务实现 |
| ScenicServiceImpl.java   | 景点和景点相册业务实现 |
| FoodServiceImpl.java     | 美食业务实现 |
| FavoriteServiceImpl.java | 收藏业务实现 |
| CommentServiceImpl.java  | 评论业务实现 |

---

### 7. resources/mapper

该目录用于存放 MyBatis 的 XML 映射文件。

主要文件：

```text
UserMapper.xml
ScenicMapper.xml
FoodMapper.xml
FavoriteMapper.xml
CommentMapper.xml
```

---

### 8. resources/static

该目录用于存放静态资源，例如景点图片、美食图片和后续可扩展的视频资源。

当前景点和美食图片由后端静态资源目录统一管理，景点图片采用“每个景点一个子目录”的方式保存：

```text
static
├── images
│   ├── scenic
│   │   ├── sunzhongshan
│   │   ├── zhanyuan
│   │   ├── qijiang
│   │   ├── zimaling
│   │   └── xiaolan
│   └── food
└── videos
```

景点封面图和相册图路径示例：

```text
/images/scenic/sunzhongshan/1.jpg
/images/scenic/sunzhongshan/2.jpg
/images/scenic/sunzhongshan/3.jpg
/images/food/shiqiruge.jpg
```

前端通过后端返回的资源路径加载图片，模拟器访问时由前端统一拼接后端服务地址。

---

### 9. resources/sql

该目录用于存放数据库初始化 SQL 文件。

主要文件：

```text
init.sql
```

该文件中可以保存建表语句和初始化测试数据。

## 六、后端分层调用流程

后端接口的调用流程如下：

```text
HarmonyOS APP
    ↓ HTTP 请求
Controller 控制层
    ↓ 调用业务方法
Service 业务层
    ↓ 调用数据库操作
Mapper 数据访问层
    ↓ 执行 SQL
MySQL 数据库
    ↓ 返回数据
Mapper
    ↓
Service
    ↓
Controller
    ↓ JSON 响应
HarmonyOS APP
```

## 九、素材资源处理方式

当前项目已准备景点图片和美食图片，后端通过 `src/main/resources/static` 对外提供静态资源访问。景点详情页的轮播图由 `scenic_image` 表维护，前端通过 `/api/scenic/images/{scenicId}` 获取图片列表。

### 1. 景点图片路径

```text
/images/scenic/sunzhongshan/1.jpg
/images/scenic/sunzhongshan/2.jpg
/images/scenic/sunzhongshan/3.jpg
/images/scenic/zhanyuan/1.jpg
/images/scenic/zhanyuan/2.jpg
/images/scenic/zhanyuan/3.jpg
/images/scenic/qijiang/1.jpg
/images/scenic/qijiang/2.jpg
/images/scenic/qijiang/3.jpg
/images/scenic/zimaling/1.jpg
/images/scenic/zimaling/2.jpg
/images/scenic/zimaling/3.jpg
/images/scenic/xiaolan/1.jpg
/images/scenic/xiaolan/2.jpg
/images/scenic/xiaolan/3.jpg
```

### 2. 美食图片路径

```text
/images/food/shiqiruge.jpg
/images/food/shaxikourou.jpg
/images/food/sanxianglaifen.jpg
/images/food/cuirouwan.jpg
/images/food/juhuayan.jpg
```

### 3. 后续替换方式

后续如果替换图片，只要保持数据库路径和文件名一致即可。景点相册图片建议继续使用每个景点一个目录、`1.jpg` 到 `3.jpg` 的命名方式：

```text
src/main/resources/static/images/scenic/sunzhongshan
src/main/resources/static/images/scenic/zhanyuan
src/main/resources/static/images/scenic/qijiang
src/main/resources/static/images/scenic/zimaling
src/main/resources/static/images/scenic/xiaolan
src/main/resources/static/images/food
```

如果后续补充视频资源，可继续放入 `src/main/resources/static/videos`，并在数据库 `video_url` 字段中保存对应路径。

## 十、后端项目阶段小结

本后端项目采用 Spring Boot + MyBatis + MySQL 技术路线，整体按照 controller、service、mapper、entity、common 等包进行分层开发。该结构清晰，适合课程设计项目，也方便后续与 HarmonyOS APP 进行接口联调。

后端开发时应优先完成用户登录注册、景点列表、景点详情、景点相册、美食列表和详情接口，再逐步实现收藏和评论功能。图片素材由后端静态资源目录统一管理，数据库保存相对路径，前端通过接口返回路径进行展示。
