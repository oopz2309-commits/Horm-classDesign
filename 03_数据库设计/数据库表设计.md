# 中山文旅美食导览 App 数据库表设计

## 一、数据库设计说明

本项目使用 MySQL 数据库进行数据管理，主要存储用户信息、景点信息、美食信息、收藏信息和评论信息。数据库设计围绕“中山文旅美食导览 App”的核心业务展开，能够支持用户登录注册、景点浏览、美食浏览、搜索、收藏和评论等功能。

数据库名称建议设置为：

```sql
zs_travel_app
```

---

## 二、数据表总览

| 序号 | 表名          | 中文名称 | 主要作用            |
| -- | ----------- | ---- | --------------- |
| 1  | user        | 用户表  | 存储用户账号、密码、昵称等信息 |
| 2  | scenic_spot | 景点表  | 存储中山景点信息        |
| 3  | food        | 美食表  | 存储中山特色美食信息      |
| 4  | favorite    | 收藏表  | 存储用户收藏的景点或美食    |
| 5  | comment     | 评论表  | 存储用户对景点或美食的评论   |

---

# 三、用户表 user

## 1. 表作用

用户表用于存储 APP 用户的基本信息，包括用户名、密码、昵称、头像和注册时间。该表主要用于登录、注册和个人中心展示。

## 2. 表结构

| 字段名         | 类型       | 长度  | 是否为空 | 主键 | 说明     |
| ----------- | -------- | --- | ---- | -- | ------ |
| id          | int      | 11  | 否    | 是  | 用户 ID  |
| username    | varchar  | 50  | 否    | 否  | 用户名    |
| password    | varchar  | 100 | 否    | 否  | 用户密码   |
| nickname    | varchar  | 50  | 是    | 否  | 用户昵称   |
| avatar      | varchar  | 255 | 是    | 否  | 用户头像地址 |
| create_time | datetime | -   | 是    | 否  | 注册时间   |

## 3. 建表 SQL

```sql
CREATE TABLE user (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '用户密码',
    nickname VARCHAR(50) DEFAULT NULL COMMENT '用户昵称',
    avatar VARCHAR(255) DEFAULT NULL COMMENT '用户头像地址',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间'
) COMMENT='用户表';
```

---

# 四、景点表 scenic_spot

## 1. 表作用

景点表用于存储中山市特色景点信息，包括景点名称、封面图片、视频地址、景点地址、景点介绍、景点类型和推荐评分。

## 2. 表结构

| 字段名         | 类型       | 长度  | 是否为空 | 主键 | 说明     |
| ----------- | -------- | --- | ---- | -- | ------ |
| id          | int      | 11  | 否    | 是  | 景点 ID  |
| name        | varchar  | 100 | 否    | 否  | 景点名称   |
| cover_image | varchar  | 255 | 是    | 否  | 景点封面图  |
| video_url   | varchar  | 255 | 是    | 否  | 景点视频地址 |
| address     | varchar  | 255 | 是    | 否  | 景点地址   |
| description | text     | -   | 是    | 否  | 景点介绍   |
| type        | varchar  | 50  | 是    | 否  | 景点类型   |
| score       | decimal  | 2,1 | 是    | 否  | 推荐评分   |
| create_time | datetime | -   | 是    | 否  | 创建时间   |

## 3. 建表 SQL

```sql
CREATE TABLE scenic_spot (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '景点ID',
    name VARCHAR(100) NOT NULL COMMENT '景点名称',
    cover_image VARCHAR(255) DEFAULT NULL COMMENT '景点封面图',
    video_url VARCHAR(255) DEFAULT NULL COMMENT '景点视频地址',
    address VARCHAR(255) DEFAULT NULL COMMENT '景点地址',
    description TEXT COMMENT '景点介绍',
    type VARCHAR(50) DEFAULT NULL COMMENT '景点类型',
    score DECIMAL(2,1) DEFAULT 5.0 COMMENT '推荐评分',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) COMMENT='景点表';
```

---

# 五、美食表 food

## 1. 表作用

美食表用于存储中山市特色美食信息，包括美食名称、封面图片、美食介绍、推荐店铺、店铺地址和人均价格。

## 2. 表结构

| 字段名            | 类型       | 长度  | 是否为空 | 主键 | 说明    |
| -------------- | -------- | --- | ---- | -- | ----- |
| id             | int      | 11  | 否    | 是  | 美食 ID |
| name           | varchar  | 100 | 否    | 否  | 美食名称  |
| cover_image    | varchar  | 255 | 是    | 否  | 美食封面图 |
| description    | text     | -   | 是    | 否  | 美食介绍  |
| recommend_shop | varchar  | 100 | 是    | 否  | 推荐店铺  |
| address        | varchar  | 255 | 是    | 否  | 店铺地址  |
| price          | decimal  | 6,2 | 是    | 否  | 人均价格  |
| create_time    | datetime | -   | 是    | 否  | 创建时间  |

## 3. 建表 SQL

```sql
CREATE TABLE food (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '美食ID',
    name VARCHAR(100) NOT NULL COMMENT '美食名称',
    cover_image VARCHAR(255) DEFAULT NULL COMMENT '美食封面图',
    description TEXT COMMENT '美食介绍',
    recommend_shop VARCHAR(100) DEFAULT NULL COMMENT '推荐店铺',
    address VARCHAR(255) DEFAULT NULL COMMENT '店铺地址',
    price DECIMAL(6,2) DEFAULT NULL COMMENT '人均价格',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) COMMENT='美食表';
```

---

# 六、收藏表 favorite

## 1. 表作用

收藏表用于存储用户收藏的景点或美食信息。由于用户既可以收藏景点，也可以收藏美食，因此使用 target_id 表示收藏对象 ID，使用 type 区分收藏类型。

## 2. 表结构

| 字段名         | 类型       | 长度 | 是否为空 | 主键 | 说明                 |
| ----------- | -------- | -- | ---- | -- | ------------------ |
| id          | int      | 11 | 否    | 是  | 收藏 ID              |
| user_id     | int      | 11 | 否    | 否  | 用户 ID              |
| target_id   | int      | 11 | 否    | 否  | 收藏对象 ID            |
| type        | varchar  | 20 | 否    | 否  | 收藏类型：scenic 或 food |
| create_time | datetime | -  | 是    | 否  | 收藏时间               |

## 3. 建表 SQL

```sql
CREATE TABLE favorite (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '收藏ID',
    user_id INT NOT NULL COMMENT '用户ID',
    target_id INT NOT NULL COMMENT '收藏对象ID',
    type VARCHAR(20) NOT NULL COMMENT '收藏类型：scenic或food',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '收藏时间'
) COMMENT='收藏表';
```

---

# 七、评论表 comment

## 1. 表作用

评论表用于存储用户对景点或美食发布的评论内容。和收藏表类似，评论对象可能是景点，也可能是美食，因此使用 target_id 表示评论对象 ID，使用 type 区分评论类型。

## 2. 表结构

| 字段名         | 类型       | 长度  | 是否为空 | 主键 | 说明                 |
| ----------- | -------- | --- | ---- | -- | ------------------ |
| id          | int      | 11  | 否    | 是  | 评论 ID              |
| user_id     | int      | 11  | 否    | 否  | 用户 ID              |
| target_id   | int      | 11  | 否    | 否  | 评论对象 ID            |
| type        | varchar  | 20  | 否    | 否  | 评论类型：scenic 或 food |
| content     | varchar  | 500 | 否    | 否  | 评论内容               |
| create_time | datetime | -   | 是    | 否  | 评论时间               |

## 3. 建表 SQL

```sql
CREATE TABLE comment (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '评论ID',
    user_id INT NOT NULL COMMENT '用户ID',
    target_id INT NOT NULL COMMENT '评论对象ID',
    type VARCHAR(20) NOT NULL COMMENT '评论类型：scenic或food',
    content VARCHAR(500) NOT NULL COMMENT '评论内容',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '评论时间'
) COMMENT='评论表';
```

# 九、数据库设计小结

本项目数据库共设计 5 张核心数据表，分别为用户表、景点表、美食表、收藏表和评论表。用户表用于支持登录注册功能；景点表和美食表用于支持 APP 的主要内容展示；收藏表用于记录用户收藏的景点和美食；评论表用于记录用户对景点和美食的评价。该数据库设计能够满足中山文旅美食导览 App 的基本业务需求。
