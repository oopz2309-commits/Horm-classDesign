# 中山文旅美食导览 App 后端接口设计

## 一、接口设计说明

本项目采用前后端分离架构，HarmonyOS APP 作为前端客户端，Spring Boot 作为后端服务，MySQL 作为数据库。APP 通过 HTTP 请求访问后端接口，后端返回 JSON 格式数据。

接口基础路径建议设置为：

```text
/api
```

统一返回数据格式建议设计为：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

其中：

| 字段      | 说明                    |
| ------- | --------------------- |
| code    | 状态码，200 表示成功，500 表示失败 |
| message | 返回提示信息                |
| data    | 返回的数据内容               |

### 接口地址

```text

```

### 功能说明

用于新用户注册账号。

### 请求参数

  "password": "123456",
  "nickname": "张三"

  "data": null

## 3. 获取景点相册接口

### 接口地址

```text
GET /api/scenic/images/{scenicId}
```

### 功能说明

根据景点 ID 获取景点详情页相册轮播图片。前端进入景点详情页后调用该接口，按照 `sortOrder` 从小到大展示轮播图。

### 路径参数

| 参数名   | 类型 | 是否必填 | 说明    |
| -------- | ---- | -------- | ------- |
| scenicId | int  | 是       | 景点 ID |

### 返回示例

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 1,
      "scenicId": 1,
      "imageUrl": "/images/scenic/sunzhongshan/1.jpg",
      "sortOrder": 1
    },
    {
      "id": 2,
      "scenicId": 1,
      "imageUrl": "/images/scenic/sunzhongshan/2.jpg",
      "sortOrder": 2
    },
    {
      "id": 3,
      "scenicId": 1,
      "imageUrl": "/images/scenic/sunzhongshan/3.jpg",
      "sortOrder": 3
    }
  ]
}
```

# 七、搜索模块接口

为了简化开发，搜索功能可以分别调用景点搜索接口和美食搜索接口。

| 搜索内容 | 接口                                 |
| ---- | ---------------------------------- |
| 搜索景点 | GET /api/scenic/search?keyword=关键词 |
| 搜索美食 | GET /api/food/search?keyword=关键词   |

如果后期想做统一搜索接口，也可以设计为：

```text
GET /api/search?keyword=关键词
```

返回示例：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "scenicList": [],
    "foodList": []
  }
}
```

# 八、接口设计小结

本项目共设计用户模块、景点模块、美食模块、收藏模块、评论模块和搜索模块接口。其中景点模块除列表、详情和搜索外，还提供景点相册接口，用于支持详情页轮播图展示。HarmonyOS APP 通过 HTTP 请求访问 Spring Boot 后端接口，后端从 MySQL 数据库查询数据并返回 JSON 格式结果。该接口设计能够满足用户登录注册、景点展示、景点相册轮播、美食展示、详情查看、搜索、收藏和评论等核心功能需求。
