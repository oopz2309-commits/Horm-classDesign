# 基于 HarmonyOS 的中山文旅美食导览 App 页面原型说明

## 一、页面原型设计说明

本项目名称为《基于 HarmonyOS 的中山文旅美食导览 App 设计与实现》。APP 以中山市特色景点、美食和本地文化为主题，面向游客和本地居民提供景点浏览、美食推荐、搜索、收藏、评论和个人中心等功能。

根据课程设计要求，本系统至少需要包含 8 个 UI 页面、4 个功能模块、登录功能、数据管理功能、多媒体技术和网络功能。本项目共设计 10 个主要页面，分别为启动页、登录页、注册页、首页、景点列表页、景点详情页、美食列表页、美食详情页、搜索页和我的页面，满足课程设计的页面数量要求。

---

## 三、底部导航栏设计

为了方便用户在主要功能模块之间切换，本 APP 在用户登录成功后使用底部导航栏进行页面导航。底部导航栏共设置 4 个主入口，分别为：首页、景点、美食、我的。

| 导航名称 | 对应页面           | 功能说明                        |
| ---- | -------------- | --------------------------- |
| 首页   | HomePage       | 展示 APP 推荐内容，包括轮播图、热门景点和推荐美食 |
| 景点   | ScenicListPage | 展示中山市特色景点列表                 |
| 美食   | FoodListPage   | 展示中山市特色美食列表                 |
| 我的   | MinePage       | 展示用户个人信息、收藏记录和退出登录入口        |

底部导航栏显示在首页、景点列表页、美食列表页和我的页面中。登录页、注册页、启动页、详情页和搜索页不显示底部导航栏。

底部导航栏页面关系如下：

```text
首页 HomePage
景点 ScenicListPage
美食 FoodListPage
我的 MinePage
```

## 四、页面跳转关系设计

APP 的整体页面跳转流程如下：

```text
启动页
  ↓
登录页
  ↓
首页
```

用户在登录页可以跳转到注册页：

```text
登录页
  ↓
注册页
  ↓
登录页
```

用户登录成功后进入首页，首页可以跳转到景点列表页、美食列表页、搜索页和我的页面：

```text
首页
  ├── 景点列表页
  ├── 美食列表页
  ├── 搜索页
  └── 我的页面
```

景点列表页可以跳转到景点详情页：

```text
景点列表页
  ↓
景点详情页
```

美食列表页可以跳转到美食详情页：

```text
美食列表页
  ↓
美食详情页
```

搜索页根据搜索结果类型，可以跳转到景点详情页或美食详情页：

```text
搜索页
  ├── 景点详情页
  └── 美食详情页
```

我的页面可以查看用户信息、我的收藏、关于系统，也可以退出登录：

```text
我的页面
  ├── 我的收藏
  ├── 关于系统
  └── 退出登录
```

# 五、具体页面原型说明

## 1. 启动页 SplashPage

### 页面作用

启动页是用户打开 APP 后首先看到的页面，主要用于展示 APP 名称、Logo、欢迎语和项目主题。该页面停留短暂时间后自动跳转到登录页。

### 页面元素

| 页面元素     | 说明                        |
| -------- | ------------------------- |
| APP Logo | 展示应用标识，可以使用中山文旅相关图片或自定义图标 |
| APP 名称   | 显示“中山文旅美食导览”              |
| 欢迎语      | 显示“发现中山之美，品味城市烟火”         |
| 背景图      | 可使用中山景点图片作为背景             |
| 自动跳转     | 停留 2 秒后跳转到登录页             |

### 页面布局

```text
------------------------------------------------
|                                              |
|                 APP Logo                     |
|                                              |
|            中山文旅美食导览                  |
|                                              |
|        发现中山之美，品味城市烟火              |
|                                              |
------------------------------------------------
```

### 交互说明

用户打开 APP 后进入启动页，系统短暂停留后自动跳转到登录页。如果用户已经登录过，也可以后期扩展为自动进入首页。

---

## 2. 登录页 LoginPage

### 页面作用

登录页用于用户输入账号和密码进行登录。登录成功后进入首页，未注册用户可以跳转到注册页。

### 页面元素

| 页面元素   | 说明              |
| ------ | --------------- |
| 页面标题   | 显示“欢迎登录”        |
| 用户名输入框 | 输入用户账号或用户名      |
| 密码输入框  | 输入用户密码          |
| 登录按钮   | 点击后进行登录验证       |
| 注册入口   | 点击跳转到注册页        |
| 忘记密码   | 可选功能，课程设计阶段可不实现 |

### 页面布局

```text
------------------------------------------------
|                  欢迎登录                     |
|                                              |
|              中山文旅美食导览                 |
|                                              |
|        用户名：[                    ]         |
|                                              |
|        密码：  [                    ]         |
|                                              |
|              [ 登录 ]                         |
|                                              |
|          没有账号？去注册                      |
------------------------------------------------
```

### 交互说明

用户输入用户名和密码后，点击登录按钮。系统调用登录接口进行验证，验证成功后跳转到首页；验证失败则提示“用户名或密码错误”。点击“去注册”可以跳转到注册页。

---

## 3. 注册页 RegisterPage

### 页面作用

注册页用于新用户创建账号。用户填写用户名、密码、确认密码和昵称后，点击注册按钮完成账号注册。

### 页面元素

| 页面元素    | 说明        |
| ------- | --------- |
| 页面标题    | 显示“用户注册”  |
| 用户名输入框  | 输入注册账号    |
| 昵称输入框   | 输入用户昵称    |
| 密码输入框   | 输入密码      |
| 确认密码输入框 | 再次输入密码    |
| 注册按钮    | 点击后提交注册信息 |
| 返回登录    | 点击返回登录页   |

### 页面布局

```text
------------------------------------------------
|                  用户注册                     |
|                                              |
|        用户名：[                    ]         |
|                                              |
|        昵称：  [                    ]         |
|                                              |
|        密码：  [                    ]         |
|                                              |
|        确认密码：[                  ]         |
|                                              |
|              [ 注册 ]                         |
|                                              |
|              已有账号？去登录                 |
------------------------------------------------
```

### 交互说明

用户填写注册信息后，点击注册按钮。系统判断两次密码是否一致，如果一致则调用注册接口保存用户信息。注册成功后提示“注册成功”，并跳转回登录页。

---

## 9. 搜索页 SearchPage

### 页面作用

搜索页用于根据关键词搜索景点和美食，方便用户快速找到自己感兴趣的内容。

### 页面元素

| 页面元素   | 说明            |
| ------ | ------------- |
| 搜索输入框  | 输入景点或美食关键词    |
| 搜索按钮   | 点击后执行搜索       |
| 搜索分类   | 可选择搜索景点或美食    |
| 搜索结果列表 | 展示搜索结果        |
| 空结果提示  | 没有搜索结果时显示提示信息 |

### 页面布局

```text
------------------------------------------------
|  < 返回                                      |
|                                              |
```

# 八、小结

本 APP 页面原型设计围绕“中山文旅美食导览”主题展开，共设计 10 个主要页面，并通过底部导航栏连接首页、景点、美食和我的四个核心页面。系统功能覆盖用户登录注册、首页推荐、景点浏览、美食浏览、搜索、收藏、评论和个人中心等内容，能够满足课程设计对页面数量、功能模块、登录、数据库、多媒体技术和网络功能的要求。
